# Secret Vaults вҖ” Design

**Date:** 2026-06-06
**Status:** Approved

## Summary

Personal web app to store secrets (API keys, connection strings, client secrets). Next.js 16 static frontend; Google Apps Script (GAS) web app as backend; Google Sheet as database. Single user, protected by a long random API token.

## Decisions (made during brainstorming)

| Decision | Choice |
|---|---|
| Encryption | **Plaintext in Sheet** (user's explicit choice; accepted risk вҖ” anyone with Sheet/account access reads everything) |
| Auth | GAS deployed "anyone can access"; every request carries an API token checked against Script Properties |
| Users | Single user (owner only) |
| Hosting | Next.js static export (`output: 'export'`), deployable to any static host or run locally |
| API style | Action-RPC over a single `doPost` (Approach A) |
| GAS code management | TypeScript in repo, deployed via clasp вҖ” no browser-editor editing |

- Frontend: Next.js 16 App Router, static export, single-page vault UI.
- Backend: one GAS `doPost` handler, action-based RPC, token check first, `LockService` around all writes (Sheet has no transactions; lock prevents concurrent-write corruption).
- All HTTP responses are 200; success/failure expressed in a response envelope (GAS web app limitation).

### CORS constraint (shapes the API)

GAS web apps cannot answer `OPTIONS` preflight. A `Content-Type: application/json` POST triggers preflight and fails. Workaround: POST with `Content-Type: text/plain` and a JSON-stringified body; GAS parses `e.postData.contents` manually. The ky client is configured accordingly.

## Data model

Google Sheet, tab `secrets`, header row, one row per secret:

| Column | Type |
|---|---|
| id | UUID (generated by GAS on create) |
| name | string, required |
| value | string, required (the secret itself) |
| type | `api_key \| connection_string \| client_secret \| other` |
| project | string (service/project grouping, e.g. "AWS prod", "Stripe") |
| environment | `dev \| staging \| prod \| -` |
| tags | comma-joined string in Sheet; `string[]` in app |
| notes | string |
| createdAt | ISO 8601 (set by GAS) |
| updatedAt | ISO 8601 (set by GAS) |

A single Zod schema `Secret` is the source of truth, used for form validation and API response parsing.

## API contract

Request (always POST, `Content-Type: text/plain`, body = JSON string):

```ts
{ token: string, action: "list" | "create" | "update" | "delete", payload?: unknown }
```

Response envelope:

```ts
{ ok: true,  data: T }
{ ok: false, error: { code: "UNAUTHORIZED" | "BAD_REQUEST" | "NOT_FOUND" | "INTERNAL", message: string } }
```

Actions:

| Action | Payload | Returns |
|---|---|---|
| `list` | вҖ” | `Secret[]` |
| `create` | secret fields (no id/timestamps) | `Secret` |
| `update` | `{ id, fields }` | `Secret` |
| `delete` | `{ id }` | `{ id }` |

- Token compared against a value stored in GAS Script Properties. Mismatch вҶ’ `UNAUTHORIZED`.
- Payload validated in GAS; invalid вҶ’ `BAD_REQUEST`. Unknown id вҶ’ `NOT_FOUND`. Uncaught exception вҶ’ `INTERNAL`.

## Frontend

- **Token gate:** Zustand store persisted to localStorage holding GAS URL + token. No token вҶ’ login form; verified by issuing a `list` call. Any later `UNAUTHORIZED` вҶ’ clear store вҶ’ back to gate.
- **Server state:** TanStack Query. Single `['secrets']` query; create/update/delete mutations invalidate it.
- **Table:** TanStack Table. Columns: name, type, project, environment, tags, masked value (`вҖўвҖўвҖўвҖў` with eye-icon reveal), copy-to-clipboard button (toast confirm), edit/delete row actions. Global text search plus column filters for type/project/environment/tags.
- **Forms:** Radix Dialog + TanStack Form + Zod for add/edit. Radix AlertDialog for delete confirmation.
- **HTTP client:** one ky instance вҖ” text/plain body, envelope parsed with Zod, `ok:false` thrown as typed `ApiError`.
- **Zustand scope:** token/URL + transient UI state (revealed-row ids) only. Server data lives exclusively in TanStack Query.
- **Styling:** Tailwind CSS v4 + Radix primitives. Responsive (table degrades gracefully on small screens).

## Error handling

- ky `afterResponse` hook converts envelope `ok:false` into thrown `ApiError(code, message)`.
- Query/mutation `onError` вҶ’ Radix Toast with the message.
- `UNAUTHORIZED` вҶ’ wipe token store вҶ’ login gate.
- GAS handler wrapped in try/catch вҶ’ `INTERNAL` envelope; payload validation failures вҶ’ `BAD_REQUEST`.
- All GAS writes inside `LockService.getScriptLock()`.

## Testing

- Vitest unit tests: Zod schemas, envelope parsing, rowвҶ”record mapping.
- GAS business logic written as pure TS functions (row mapping, validation, action dispatch) so it is testable without the GAS runtime; thin GAS-API shell around them.
- API client tested against mocked fetch.
- No e2e in v1 (YAGNI).

## Out of scope (v1)

- Client- or server-side encryption (explicitly declined)
- Multiple users / sharing
- Secret versioning / audit log
- Import/export
- E2E tests
